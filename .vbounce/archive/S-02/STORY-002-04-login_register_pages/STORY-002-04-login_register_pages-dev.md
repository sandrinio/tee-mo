## Summary

This story wired up the full auth UI surface: login page, register page, protected `/app` placeholder, `ProtectedRoute` guard, and `SignOutButton`. The landing page CTA was enabled by wrapping it in a `<Link to="/login">`. All code follows the §3.3 verbatim blocks with token discipline enforced by ADR-022 (no new `@theme` tokens, design-system primitives only).

The one non-trivial implementation wrinkle was the `routeTree.gen.ts` chicken-and-egg: the `npm run build` script runs `tsc -b && vite build`, meaning TypeScript compiles before Vite gets to regenerate the route tree. On first run, `tsc -b` fails because the new routes aren't in `routeTree.gen.ts` yet. The fix is to run `vite build` first (which calls the TanStackRouterVite plugin and regenerates the tree), then the full `npm run build` passes cleanly on the second invocation. The task file anticipated this and flagged `tsr generate` as a fallback — the Vite-first approach worked.

## Process Feedback

- The build script ordering (`tsc -b && vite build`) creates a chicken-and-egg: tsc runs before Vite regenerates routeTree.gen.ts, so the first `npm run build` after adding new route files always fails. The workaround (run `vite build` first, then full `npm run build`) works cleanly, but the story task file could explicitly call this out as a step rather than just mentioning it as a fallback. Worth flagging for FLASHCARDS.md.
- Story §3.3 code blocks were accurate and complete — using them verbatim saved significant implementation time.
