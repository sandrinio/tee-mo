# STORY-014-02: Multipart Upload Endpoint

**Complexity: L2** — One new route in `knowledge.py`, one new request model, one new helper that wraps `extraction_service` + `document_service.create_document(source='upload', ...)`. Validation logic mirrors the existing Drive-index route. Blocked by STORY-014-01 (needs `extraction_service`).

---

## 1. The Spec (The Contract)

### 1.1 User Story
As a **Workspace Admin**, I want to upload a local PDF / DOCX / XLSX / TXT / MD file directly to my workspace's knowledge base, so that I can index documents that aren't in Google Drive without first uploading them there.

### 1.2 Detailed Requirements
- **R1.** New route: `POST /api/workspaces/{workspace_id}/documents/upload`. Multipart form, single field `file: UploadFile`. Authenticated via the same `get_current_user_id` dependency as sibling routes.
- **R2.** Validations, in this order, each returning the listed status code on failure:
  1. **401** — no valid auth (handled by dependency).
  2. **404** — workspace not owned by current user (`_assert_workspace_owner`).
  3. **400 "BYOK key required"** — workspace has no `ai_provider` + `encrypted_api_key`.
  4. **400 "File size exceeds 10MB limit"** — uploaded bytes > 10 × 1024 × 1024.
  5. **400 "Unsupported file type"** — `file.content_type` not in the upload allowlist (see §3.2 R-MIME).
  6. **400 "Maximum 100 files per workspace. Remove a file before adding another."** — current count of `teemo_documents` rows for this workspace ≥ 100. Read inside the per-workspace `asyncio.Lock` to prevent the same race the Drive-index path guards against. Match the existing Drive-route literal at `backend/app/api/routes/knowledge.py:228` exactly (status code + detail string) so the frontend can pattern-match on it.
  7. **409 "File already uploaded"** — a row already exists with `(workspace_id, original_filename, source='upload')`.
- **R3.** On success:
  1. Read bytes into memory once (single `await file.read()`).
  2. Dispatch to `extraction_service` based on MIME — PDF→`extract_pdf`, DOCX→`extract_docx`, XLSX→`extract_xlsx`, TXT/MD→UTF-8 decode with `latin-1` fallback. Apply `maybe_truncate`.
  3. Call `document_service.create_document(supabase, workspace_id, title=<filename>, content=<extracted>, doc_type=<mapped>, source='upload', original_filename=<filename>)`.
  4. Return the created row dict (HTTP 201).
- **R4.** The raw bytes are never written to disk, never streamed to Supabase Storage, never logged. Only the extracted text reaches the database.
- **R5.** AI description is generated by `document_service.create_document` itself (it already calls `scan_service.generate_ai_description` with the workspace BYOK key) — no new scan call is required in this story. The `sync_status` field defaults to `'pending'` (set by `create_document`); upload rows flow through `wiki_ingest_cron` exactly like Drive rows. The agent sees them in `## Available Documents` immediately because `agent.py:1004` selects all rows for the workspace without a `sync_status` filter — `pending` rows render with the `⏳` annotation. Do **not** override `sync_status` in this route.
- **R6.** `original_filename` is the exact filename Slack/the browser provided (no sanitization beyond `os.path.basename` to strip any path component). Title falls back to `original_filename` when no override is supplied.
- **R7.** Per-workspace `asyncio.Lock` (existing `_get_workspace_lock`) wraps steps R2.6 → R2.7 → R3.3 atomically.

### 1.3 Out of Scope
- Multiple-file upload, folder upload, drag-drop preview — single file per request.
- Image / audio / video uploads.
- OCR for image-only PDFs (Drive multimodal fallback is Drive-only).
- Re-uploading the same filename to overwrite — caller must DELETE first.
- Frontend UI work — STORY-014-03.
- Renaming `original_filename` post-upload.

### 3.2 Technical Logic

**R-MIME — Upload allowlist + doc_type mapping:**

| Content-Type | doc_type | Extractor |
|---|---|---|
| `application/pdf` | `pdf` | `extract_pdf` |
| `application/vnd.openxmlformats-officedocument.wordprocessingml.document` | `docx` | `extract_docx` |
| `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` | `xlsx` | `extract_xlsx` |
| `text/plain` | `text` | UTF-8 decode (fallback `latin-1`) |
| `text/markdown` | `markdown` | UTF-8 decode (fallback `latin-1`) |

Google MIME types (`application/vnd.google-apps.*`) are intentionally **excluded** from the upload allowlist — those flow only through the Drive-index route.

**Concurrency:** Reuse `_get_workspace_lock(workspace_id)` from `knowledge.py:92` — the count-then-insert race is the same one the Drive route already guards.

**Filename sanitization:** `original_filename = os.path.basename(file.filename or "untitled")`. Reject empty / null filename with 400.

**Size enforcement:** Stream-safe — call `await file.read()` once, then `len(bytes_)`. The 10MB ceiling is below FastAPI's default body-size limit, so no Starlette config change.

**Duplicate check:** `supabase.table("teemo_documents").select("id").eq("workspace_id", wid).eq("original_filename", fname).eq("source", "upload").limit(1).execute()`. If non-empty → 409.

### 3.3 API Contract

| Endpoint | Method | Auth | Request Shape | Response Shape |
|---|---|---|---|---|
| `/api/workspaces/{workspace_id}/documents/upload` | POST | Cookie / Bearer | `multipart/form-data` with field `file: UploadFile` | 201 — full `teemo_documents` row dict (id, title, source='upload', doc_type, original_filename, content, ai_description, sync_status, created_at, ...) |

Error responses match §1.2 R2 status codes + detail strings exactly so the frontend can match on them.
