```sql
CREATE TABLE IF NOT EXISTS teemo_mcp_servers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workspace_id UUID NOT NULL REFERENCES teemo_workspaces(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  transport TEXT NOT NULL CHECK (transport IN ('sse', 'streamable_http')),
  url TEXT NOT NULL,
  headers_encrypted JSONB NOT NULL DEFAULT '{}'::jsonb,
  is_active BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (workspace_id, name)
);
CREATE INDEX IF NOT EXISTS idx_teemo_mcp_servers_workspace_id
  ON teemo_mcp_servers (workspace_id);
DO $$ BEGIN RAISE NOTICE 'teemo_mcp_servers ready (% rows)', (SELECT count(*) FROM teemo_mcp_servers); END $$;
```

## Cross-story risks

1. **`AgentDeps` field order**: 012-03 appends `mcp_servers`. Any test (or future story) that builds `AgentDeps` with positional args breaks if the field is inserted in the middle. Append AT END (after `sender_tz`). 012-03 must verify no existing fixture passes more than 3 positional args.
2. **`is_safe_url` return-shape change ripples**: 012-01 changes `_is_safe_url(url) -> bool` to `is_safe_url(url) -> tuple[bool, str | None]`. Only one existing caller (`agent.py:1306`), but the rejection-message string must stay byte-identical for `http_request` regression. **012-01 owns this fix; 012-02 and 012-03 do NOT touch this seam** — they consume the already-lifted helper.
3. **`TEEMO_TABLES` ordering**: 012-01 says it appends; 012-02 §3.1 also says it appends. Whichever lands second observes the table already in the tuple. Both stories must use idempotent append (or 012-02 simply omits the change since 012-01 owns it). Recommend: **012-01 owns the `TEEMO_TABLES` line; 012-02 only mounts the router.**
4. **Pydantic AI `tools/list` API method name**: STORY-012-01 §3.2 hedges. The Developer must verify the actual method name (`list_tools()` vs `tools.list()` vs other) on the first run, then record a flashcard tagged `#mcp #pydantic-ai`. 012-02's test-connection wrapper and 012-03's tool-registration both depend on this being correct.
5. **41-tool token bloat**: Per SPRINT-17 §5, GitHub MCP registers 41 tools (~6–10k system-prompt tokens). V1 accepts the cost. **Reporter must record a flashcard at sprint close** — V2 mitigation is `X-MCP-Toolsets` filtering header in the modal "Advanced" panel.
6. **HTTP 200 on `/test` failure** (012-02): Frontend (012-04) must read `body.ok`, NOT the HTTP status code, to decide green/red. If 012-02 accidentally maps `ok=false` to 4xx/5xx, 012-04's `useTestMcpServerMutation.onError` fires instead of `onSuccess` — wrong code path. 012-04 test #13 (badge red on `ok=false`) catches this.
